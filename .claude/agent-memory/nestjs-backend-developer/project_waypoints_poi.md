The Waypoints POI feature (branch `feature/waypoints-poi`, extended in `feature/PILLY-POI-001-accomodation-sights`) adds:

- `slug` field (unique, NOT NULL) on `CaminoPoint` — generated by `CaminosService.generateSlug()` at create time, immutable thereafter
- `Accommodation` and `Sight` models linked to `CaminoPoint` via `camino_point_id` (CASCADE delete)
- `WaypointsModule`: `GET /waypoints/:slug` (public, no longer includes accommodations/sights inline), `POST /waypoints/:slug/accommodations|sights` (pilgrim role)
- `AccommodationsModule` (new, `src/accommodations/`): `GET /accommodations?caminoPointId=:id`, `GET /accommodations/:id`, `PATCH /accommodations/:id`, `DELETE /accommodations/:id`
- `SightsModule` (new, `src/sights/`): `GET /sights?caminoPointId=:id`, `GET /sights/:id`, `PATCH /sights/:id`, `DELETE /sights/:id`
- `UploadsModule`: `POST /uploads/images` (pilgrim role, multer, Supabase Storage, 10 MB × 10 files). Additional methods added in PILLY-CAM-003: `uploadImage(key, file)` (caller-controlled key, no filename appended), `deleteImageStrict(url)` (throws BadGatewayException on S3 failure — used where S3 failure must block DB delete). `deleteImages` now logs a warning when URL prefix mismatch causes keys to be skipped.

**Accommodation new fields (PILLY-POI-001):** `type` (AccommodationType enum, required, DEFAULT hostel), `email`, `website`, `addressStreet`, `addressZip`, `addressCity`, `addressCountry`, `priceRange` (PriceRange enum, optional)

**Sight new fields (PILLY-POI-001):** `address`, `latitude` (Float), `longitude` (Float)

**Key design decisions:**
- `WaypointDetailDto` no longer contains `accommodations[]` or `sights[]` arrays — those are fetched via the dedicated modules
- `updatedAt` must be set explicitly (`updatedAt: new Date()`) in every `update()` call — neither Accommodation nor Sight has `@updatedAt` in the Prisma schema
- `assertValid()` method on UpdateDto classes encapsulates cross-field validation (empty body, imageUrls/removeImageUrls mutual exclusion, lat/lon pairing) — called in service before DB access
- Role check: only `r.key === 'pilgrim'` checked — never `owner`
- WaypointsModule does NOT import AccommodationsModule or SightsModule

**Why:** Separating read/update/delete for accommodations and sights from WaypointsModule gives the frontend a clean REST interface without coupling waypoint fetch to POI fetch.
**How to apply:** When adding new fields to Accommodation or Sight, extend both the create DTO in WaypointsModule and the detail/update DTOs in AccommodationsModule or SightsModule.

New Prisma enums: `AccommodationType` (hostel, monastery, b_and_b, hotel, apartment, private_room), `PriceRange` (budget, moderate, comfortable, luxury).

After checkout: run `yarn workspace @pillyway/backend prisma:migrate:dev --name <description>` then `yarn workspace @pillyway/backend prisma generate`.
